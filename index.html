Projekt

A Bloody Rubynts (BR) egy ERC-20 token a Polygon hálózaton, célja a decentralizált gazdasági rendszer támogatása és a közösségi értékteremtés.

Token információk

  • Token név: Bloody Rubynts (BR)
  • Teljes mennyiség: 100,000,000,000 BR
  • Hálózat: Polygon
  • Decimals: 18
  • Contract: 0x3005ef86386858a31d76f4cef03d435Ad86A8E88

Hogyan vásárolj BR-t

1) Tárca (MetaMask)

Telepítsd a MetaMask-ot, hozz létre tárcát, és a seed phrase-t soha ne oszd meg.

2) Polygon hálózat

Válts Polygon hálózatra. A tranzakciók díjához szükséged lesz POL-ra.

3) QuickSwap / Uniswap

Nyisd meg a DEX-et Polygonon, csatlakoztasd a tárcád, és válaszd a BR tokent.

4) Swap POL/WPOL → BR

Kis összeggel tesztelj először. Tranzakció ellenőrzése Polygonscan-en.

Biztonsági tipp: Mindig ellenőrizd a contract címet Polygonscan-en, és csak hivatalos linkeket használj.

One-Pager (Quick Overview)

BLOODY RUBYNTS — One-Pager

What it is
Bloody Rubynts is an early-stage project built on one principle: trust is earned through actions, not promises.

Why it exists
“Bloody” represents real struggle and hard work. “Ruby” represents value that forms over time—under pressure, through patience.

Mission
To bring value back to people in a human, transparent, and verifiable way—without hype and without shortcuts.

How we approach support
We aim for gradual, accountable steps. Support is targeted and problem-based (not uncontrolled cash). Where possible, help is delivered directly to the provider (school, utility, medical institution, studio).

Focus areas (long-term direction)

  • education access
  • basic stability (housing / essential needs)
  • health-related support
  • emerging creators—especially music (studio time, mentoring, essential tools)

Token philosophy
The token is a tool, not the goal. It may support transparency, community involvement, and long-term alignment—without financial promises.

Transparency
We document actions and outcomes (with privacy respected). We share what is real—including lessons when something does not work as planned.

Bottom line
No campaigns. No guarantees. Only steady progress and honest reporting.

This document describes principles and intention, not financial advice or profit expectations.

Weekly Summary #9 – Wallet + Withdraw System Added (Manual Payout) + Admin Panel

This week we shipped a major new foundation: the Wallet and a manual BR withdrawal system with a clear payout process. The goal was to keep it simple, trackable, and safe while we continue testing the live trial build. Every withdrawal creates a request ID and can be emailed to support for a clean audit trail.

What’s new this week:

  • Wallet modal implemented (connect/disconnect, dev-mode linking, Polygon chain check).
  • Manual withdrawal requests added with a minimum payout of 100 BR.
  • Payout message included: “Payout within 24–48 hours.” (visible in the wallet UI).
  • Request ID + Email flow (copy request + mailto to rubyntspayments@gmail.com).
  • Admin panel added (internal use) to review withdrawal requests.
  • Admin dashboard stats: pending requests, total pending BR, total paid BR, all-time requested.
  • Export tools: JSON export + CSV export for record keeping.
  • Admin mode is local-only via localStorage flag (br_admin), so it’s not visible to normal players.

Notes / Next steps:

  • Run a few real-world withdrawal tests end-to-end and validate the tracking flow.
  • Decide whether to close the trial build after ~8 days and evaluate results (analytics + community feedback).
  • Continue polishing UX on mobile and ensure all modals behave consistently across screens.
